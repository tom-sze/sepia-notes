GitHub Guide

Step 1. Create a new repository

Help

Please avoid using an existing (non-empty) GitHub repository (existing files may be overwritten). Once you finish creating the repository, please paste the http (not ssh) URL into Sepia Notes' settings.

Step 2. Create a GitHub personal access token

Help

Both a classic token, and a fine-grained token will work. However, you can't set fine-grained tokens to never expire, so you will have to re-create it at least once a year. Whichever token you choose, please ensure that Sepia Notes has the permission to fetch and push.

Step 3. Copy the token into Sepia Notes settings

You can now copy and paste your token into Sepia Notes' settings, and press the "save" button. If you plan on backing up another device's notes (to a different repository under the same GitHub account) in the future, please save a copy of this token. Otherwise, you will have to create a new one.

Please be advised that we are in the process of simplifying the connection to GitHub (login with GitHub), and we appologise for any undue frustration this procedure may have caused you.
